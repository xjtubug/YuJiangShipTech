# YuJiang ShipTechnology - B2B Enterprise Website

## Overview

Complete B2B enterprise website for marine equipment manufacturer yujiangshiptech.com. Built with Next.js 14, featuring multi-language support, B2B inquiry system, admin dashboard with visitor tracking and lead scoring.

## Tech Stack

- **Framework**: Next.js 14 (App Router)
- **Database**: Prisma ORM + SQLite
- **Styling**: Tailwind CSS + Framer Motion
- **i18n**: next-intl (English, Chinese, Japanese, Arabic)
- **State Management**: Zustand
- **Charts**: Recharts
- **PDF Generation**: jsPDF + jspdf-autotable
- **Email**: Nodemailer
- **Icons**: Lucide React

## Features

### Pages

- Homepage (Hero, Featured Products, Advantages, Case Studies, Certifications, News, Search)
- Product List (Category/parameter filtering, sorting, pagination)
- Product Detail (Specs, PDF download, comparison, video, reviews, expert recommendations)
- About Us (Company intro, factory showcase, team, certifications, 360° tour placeholder)
- Contact Us (Map, form, contact information)
- Request Quote (Multi-product RFQ form with file upload)

### B2B Inquiry System

- Multi-product inquiry cart

### Multi-Language & Currency

- 4 languages: English (default), Chinese, Japanese, Arabic (RTL)

### Admin Panel (/admin)

- Lead Scoring System (Cold/Warm/Hot/Very Hot)
- Inquiry management with status workflow
- Product management
- Report generation (daily/weekly/monthly)
- CSV export functionality

### SEO & Analytics

- Dynamic sitemap.xml
- JSON-LD structured data (Product, Organization, BreadcrumbList)
- Google Analytics 4 integration
- Facebook Pixel
- LinkedIn Insight Tag
- Google Ads Remarketing

### Social Integration

- WhatsApp floating chat button

- Share buttons on product/blog pages

### GDPR Compliance

- Cookie consent banner
- Tracking scripts load only after consent
- Privacy-first approach

## Getting Started

### Prerequisites

- Node.js 18+
- npm or yarn

### Installation

```bash
# Clone the repository
git clone <repo-url>
cd YuJiangShipTech

# Install dependencies
npm install

# Set up the database
npx prisma db push

# Seed sample data
npx tsx prisma/seed.ts

# Start development server
npm run dev
```

Visit http://localhost:3000

### Environment Variables

Create a `.env` file in the root directory:

```env
# Database
DATABASE_URL="file:./dev.db"

# SMTP Configuration (for email notifications)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-email@gmail.com
SMTP_PASS=your-app-password
SALES_EMAIL=sales@yujiangshiptech.com

# Analytics (optional - leave as placeholder to disable)
NEXT_PUBLIC_GA_ID=G-XXXXXXXXXX
NEXT_PUBLIC_FB_PIXEL_ID=XXXXXXXXXXXXXXX
NEXT_PUBLIC_LINKEDIN_PARTNER_ID=XXXXXXX
NEXT_PUBLIC_GOOGLE_ADS_ID=AW-XXXXXXXXXX

# Site Configuration
NEXT_PUBLIC_SITE_URL=http://localhost:3000
NEXT_PUBLIC_WHATSAPP_NUMBER=+8613800138000

# Admin
ADMIN_SECRET=change-this-secret-key
```

## API Routes

| Route | Method | Description |
|---|---|---|
| `/api/products` | GET | List products (filter, search, sort, paginate) |
| `/api/products/[id]` | GET | Product detail |
| `/api/search` | GET | Global product search |
| `/api/inquiry` | POST | Submit RFQ inquiry |
| `/api/inquiry/[id]/pdf` | GET | Download quotation PDF |
| `/api/contact` | POST | Submit contact form |
| `/api/track` | POST | Track visitor behavior |
| `/api/admin/stats` | GET | Dashboard statistics |
| `/api/admin/visitors` | GET | Visitor list |
| `/api/admin/visitors/[id]` | GET | Visitor detail |
| `/api/admin/inquiries` | GET/PATCH | Manage inquiries |
| `/api/admin/reports` | POST | Generate reports |
| `/api/sitemap` | GET | XML sitemap |

## Lead Scoring System

| Score Range | Label | Description |
|---|---|---|
| 0-20 | Cold | New/casual visitor |
| 21-50 | Warm | Showing interest |
| 51-80 | Hot | Actively evaluating |
| 81+ | Very Hot | Ready to buy |

### Scoring Criteria

- Page view: +1 per view
- Product view: +3 per product
- Time on site: +1 per 30s (max +10)
- Inquiry submitted: +20
- Return visit: +5
- Target country: +10
- Multiple products viewed: +5
- PDF downloaded: +5

## Production Deployment

1. Set all environment variables for production
2. Use PostgreSQL or MySQL instead of SQLite for production
3. Configure proper SMTP credentials
4. Set real analytics tracking IDs
5. Build and start:

```bash
npm run build
npm run start
```
