## 系统概述

YuJiang ShipTech 是一个专业的 B2B 船用设备销售网站，提供多语言支持（中文、英文、日文、阿拉伯文），涵盖产品展示、询价管理、报价系统、订单管理、客户CRM、邮件营销、新闻资讯、数据分析等完整的 B2B 电商功能。

## 本地开发环境搭建

### 前置条件

- Node.js >= 18.x
- MySQL 8.x
- MinIO（对象存储，可选）
- npm 或 yarn

### 安装步骤

```bash
# 1. 克隆项目
git clone <repo-url>
cd YuJiangShipTech

# 2. 安装依赖
npm install

# 3. 配置环境变量
cp .env.example .env
# 编辑 .env 文件，配置以下必要变量：
# - DATABASE_URL：MySQL连接字符串
# - NEXTAUTH_SECRET：认证密钥
# - SMTP_*：邮件服务配置
# - MINIO_*：对象存储配置

# 4. 初始化数据库
npx prisma db push
# 或者使用迁移
npx prisma migrate dev

# 5. 生成 Prisma 客户端
npx prisma generate

# 6. 填充测试数据（可选）
npm run seed

# 7. 启动开发服务器
npm run dev
```

访问 http://localhost:3000 即可查看网站。

### 部署步骤

#### 1. 服务器初始化

```bash
# Ubuntu 22.04/24.04 推荐
sudo apt update && sudo apt upgrade -y

# 安装 Node.js 20.x
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# 安装 MySQL 8
sudo apt install -y mysql-server
sudo mysql_secure_installation

# 安装 Nginx
sudo apt install -y nginx

# 安装 PM2（进程管理）
sudo npm install -g pm2

# 安装 MinIO（可选，也可用 S3）
wget https://dl.min.io/server/minio/release/linux-amd64/minio
chmod +x minio
sudo mv minio /usr/local/bin/
```

#### 2. 数据库配置

```bash
# 登录 MySQL
sudo mysql

# 创建数据库和用户
CREATE DATABASE YuJiangShipTech CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'yujiang'@'localhost' IDENTIFIED BY 'strong-password-here';
GRANT ALL PRIVILEGES ON YuJiangShipTech.* TO 'yujiang'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

#### 3. 项目部署

```bash
# 克隆项目
cd /var/www
git clone <repo-url> yujiangshiptech
cd yujiangshiptech

# 安装依赖
npm ci --production=false

# 配置环境变量
cp .env.example .env
nano .env
# 修改 DATABASE_URL, NEXTAUTH_SECRET, SMTP 配置等

# 推送数据库结构
npx prisma db push

# 生成 Prisma 客户端
npx prisma generate

# 构建项目
npm run build

# 初始化数据（首次）
npm run seed
```

#### 5. Nginx 反向代理

```nginx
# /etc/nginx/sites-available/yujiangshiptech
server {
    listen 80;
    server_name your-domain.com www.your-domain.com;

    # 重定向到 HTTPS
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com www.your-domain.com;

    # SSL 证书
    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

    # SSL 配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # Gzip 压缩
    gzip on;
    gzip_types text/plain application/json application/javascript text/css;

    # 静态资源缓存
    location /_next/static {
        proxy_pass http://localhost:3000;
        expires 365d;
        add_header Cache-Control "public, immutable";
    }

    location /public {
        proxy_pass http://localhost:3000;
        expires 30d;
    }

    # 反向代理
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

#### 7. 自动化备份

```bash
# 创建备份脚本
cat > /var/www/backup.sh << 'EOF'
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR=/var/backups/yujiang

mkdir -p $BACKUP_DIR

# 数据库备份
mysqldump -u yujiang -p'password' YuJiangShipTech | gzip > $BACKUP_DIR/db_$DATE.sql.gz

# MinIO 数据备份
tar -czf $BACKUP_DIR/minio_$DATE.tar.gz /data/minio

# 保留最近30天
find $BACKUP_DIR -mtime +30 -delete

echo "Backup completed: $DATE"
EOF

chmod +x /var/www/backup.sh

# 添加定时任务（每日凌晨3点备份）
(crontab -l 2>/dev/null; echo "0 3 * * * /var/www/backup.sh >> /var/log/backup.log 2>&1") | crontab -
```

## 常见问题

### Q: 构建报错 "different slug names for the same dynamic path"
**A:** 删除冲突的空 `[slug]` 目录，保留 `[id]` 目录。已在本次优化中彻底修复。

### Q: 图片上传失败
**A:** 检查 MinIO 是否正常运行，确认 `.env` 中 MINIO_* 配置正确。

### Q: 邮件发送失败
**A:** 检查 SMTP 配置，Gmail 需使用应用密码而非账户密码。

### Q: 数据库连接失败
**A:** 确认 MySQL 服务已启动，DATABASE_URL 格式正确。

### Q: 多语言不生效
**A:** 确保 `src/messages/` 下对应语言的 JSON 文件已更新。

## 维护建议

1. **定期备份**：数据库每日自动备份，MinIO 文件每周备份
2. **监控告警**：使用 PM2 监控进程，设置邮件告警
3. **安全更新**：每月检查依赖安全漏洞 `npm audit`
4. **性能监控**：使用 Next.js Analytics 或 Vercel Speed Insights
5. **日志轮转**：配置 PM2 日志轮转，避免磁盘空间占满
